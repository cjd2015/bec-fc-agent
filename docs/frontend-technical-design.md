# BEC 商务英语智能学习平台 - 前端技术设计文档

**版本:** 1.0  
**更新日期:** 2026-03-31  
**适用范围:** Web/H5 学习端、内容管理端、审核管理端

---

## 1. 文档目标

本文档用于定义 BEC 商务英语智能学习平台前端部分的技术架构、模块划分、页面结构、状态管理、接口调用方式与部署策略，指导前端研发落地。

---

## 2. 前端定位

前端承担以下职责：
- 提供用户学习主界面
- 提供内容管理与审核后台界面
- 承接水平测试、单词学习、句型学习、场景训练、模拟考试等核心交互流程
- 调用阿里云函数计算 FC 提供的后端 API

前端不承担：
- 业务规则计算
- 数据持久化
- AI 推理逻辑

---

## 3. 技术选型

### 3.1 核心技术栈
- React 18
- TypeScript
- Vite
- React Router
- Zustand
- React Query
- Ant Design 5

### 3.2 选型原因
- React 适合复杂交互型业务系统
- TypeScript 有利于大型项目维护和接口约束
- Vite 构建速度快，适合前期快速迭代
- Ant Design 适合学习端与管理端并存场景
- Zustand 轻量，适合用户态和局部状态管理
- React Query 适合服务端数据缓存与异步请求管理

## 7. API 调用策略

### 7.1 接口访问方式
- 前端统一通过 `/api/` 路径访问后端
- ECS 上的 Nginx 负责将 `/api/` 转发到阿里云函数计算 FC

### 7.2 请求规范
- 统一封装 request 层
- 统一处理 token 注入
- 统一处理错误提示
- 统一处理登录失效跳转

### 7.3 典型接口分组
- `/api/v1/auth/*`
- `/api/v1/level-test/*`
- `/api/v1/vocab/*`
- `/api/v1/patterns/*`
- `/api/v1/scenes/*`
- `/api/v1/mock-exams/*`
- `/api/v1/content/*`
- `/api/v1/review/*`

## 8. 前端部署方案

### 8.1 构建方式
- 前端代码在本地或 ECS 上执行 `npm run build`
- 生成静态产物目录（如 `dist/`）

### 8.2 部署方式
优先方案：
- 将 `dist/` 部署到 ECS 的 Nginx 静态目录

备选方案：
- 部署到 OSS 静态站点

### 8.3 域名接入
- 主域名 `datahive.top` 指向 ECS
- Nginx 配置 HTTPS
- `/` 提供前端静态资源
- `/api/` 反向代理到 FC

## 9. 风险与约束

### 9.1 风险
- 页面过多导致开发节奏失控
- 学习端和管理端交互差异大，组件复用困难
- 场景训练页交互复杂，容易成为开发瓶颈

### 9.2 应对建议
- 首版优先做学习主链路页面
- 管理后台先做最小可用版本
- 场景训练页先做文本交互，不做语音

## 10. 实施顺序建议

### 第一阶段
- 登录/注册
- 水平测试
- 单词学习
- 句型学习
- 场景训练
- 模拟考试

### 第二阶段
- 学习记录与看板
- 内容管理与审核后台

### 第三阶段
- 收藏/复习
- 推荐页面
- 积分与激励页

---

## 11. 结论

前端应采用 React + TypeScript 的标准工程化方案，通过 ECS 上的 Nginx 承载静态资源，统一以 `/api/` 路径访问部署在阿里云函数计算 FC 的后端 API，实现前后端分层部署与低成本落地。
