# 前后端安全检查清单

**版本:** 1.0  
**更新日期:** 2026-03-31

## 1. 前端安全检查
### XSS 防护
- [ ] 所有用户输入默认按纯文本渲染
- [ ] 禁止随意使用 `dangerouslySetInnerHTML`
- [ ] 如必须渲染 HTML，先做白名单过滤
- [ ] 富文本内容做 HTML Sanitization
- [ ] URL 参数输出前做转义

### CSS 注入 / 样式注入
- [ ] 不允许把用户输入直接拼进 `style`
- [ ] 不允许把不可信字符串直接当 className / style 使用
- [ ] 避免动态拼接不受控内联样式
- [ ] 外部主题配置必须做白名单限制

### iframe / 点击劫持
- [ ] 配置 `X-Frame-Options: SAMEORIGIN` 或 `DENY`
- [ ] 配置 CSP 的 `frame-ancestors`
- [ ] 不允许站点被未知第三方 iframe 嵌套

### 前端敏感信息
- [ ] 不在前端代码中硬编码密钥
- [ ] 不暴露数据库地址、密码、内部服务地址
- [ ] 不在构建产物中包含 `.env` 敏感变量
- [ ] Token 不打印到 console

## 2. 后端安全检查
### SQL 注入防护
- [ ] 必须使用 ORM 或参数化查询
- [ ] 禁止字符串拼接 SQL
- [ ] 查询条件做白名单校验
- [ ] 排序字段、筛选字段不允许前端任意透传拼接

### JWT Token 安全
- [ ] JWT secret 使用强随机字符串
- [ ] 不使用默认 secret
- [ ] Token 设置过期时间
- [ ] 校验签名、过期时间、用户状态
- [ ] 不把敏感数据写进 JWT payload

### 输入校验
- [ ] 所有接口参数必须校验
- [ ] 字符串长度限制
- [ ] 枚举值限制
- [ ] 文件上传类型限制
- [ ] URL 参数合法性检查

### 权限控制
- [ ] 所有受保护接口必须鉴权
- [ ] 管理接口与普通用户接口分离
- [ ] 审核/发布接口必须做权限校验
- [ ] 用户只能访问自己的数据

### 敏感数据保护
- [ ] 用户密码必须加密存储（bcrypt）
- [ ] 数据库密码不写入代码仓库
- [ ] 模型 API Key 不写入前端
- [ ] 敏感环境变量通过部署环境注入
- [ ] 日志中不打印密码、Token、Key、连接串

## 3. 跨域 / CORS 安全检查
- [ ] CORS 只允许可信域名
- [ ] 不允许 `*` 与凭证同时使用
- [ ] 只开放必要方法和头部

## 4. FC / 网关暴露面安全检查
- [ ] FC HTTP Trigger 不允许使用 `anonymous`
- [ ] FC 直连公网 URL 不作为正式对外 API 入口
- [ ] 外部流量统一经 `datahive.top` + ECS Nginx / 网关进入
- [ ] FC 认证信息仅保存在服务端，不下发前端
- [ ] 若使用 FC Function Auth，调用方必须安全保存调用凭证
- [ ] 对 FC 直连地址进行定期暴露面检查，避免被绕过主站入口

## 5. 常见安全响应头
- [ ] `X-Frame-Options`
- [ ] `X-Content-Type-Options: nosniff`
- [ ] `Referrer-Policy`
- [ ] `Permissions-Policy`
- [ ] `Content-Security-Policy`

## 5. 文件上传安全
- [ ] 限制文件类型
- [ ] 限制文件大小
- [ ] 重命名存储
- [ ] 不信任原始文件名
- [ ] 不允许上传可执行脚本文件
- [ ] 上传目录与运行目录隔离

## 6. 日志与审计安全
- [ ] 登录失败可记录
- [ ] 审核/发布操作留痕
- [ ] 不记录敏感明文
- [ ] 异常日志可追踪但不泄露密钥

## 7. 数据库安全
- [ ] 最小权限账号
- [ ] 应用账号不使用 root
- [ ] 数据库端口不全网暴露
- [ ] 定期备份
- [ ] 备份文件加访问控制
