# BEC 商务英语智能学习平台 - 需求文档

**版本:** 2.0  
**更新日期:** 2026-03-31  
**产品定位:** 面向 BEC 备考与商务英语能力提升的智能学习平台

---

## 1. 项目概述

构建一个面向 **大学生、商务人士以及对 BEC 感兴趣用户** 的商务英语智能学习平台，围绕 **水平测试、单词学习、句型学习、场景对话训练、模拟考试、学习反馈** 构建完整学习闭环，帮助用户提升 BEC 应试能力和真实商务英语表达能力。

### 1.1 核心目标

- ✅ 支持用户进行商务英语水平测试
- ✅ 提供 BEC 场景化单词与句型学习
- ✅ 支持日常/商务场景对话训练
- ✅ 提供模拟考试与阶段性评估
- ✅ 实现学习记录、反馈与后续推荐基础能力
- ✅ 建立商务英语数据采集与内容生产体系

---

## 2. 目标用户

### 2.1 大学生
- 有 BEC 初级/中级/高级备考需求
- 需要清晰的学习路径和题型训练
- 更关注通过率和系统性备考

### 2.2 有工作经验的商务人士
- 需要提升会议、邮件、谈判、汇报等商务英语表达能力
- 学习时间碎片化，更关注实用性与效率

### 2.3 对 BEC 感兴趣的人
- 对商务英语和 BEC 考试有兴趣
- 需要低门槛测试、入门路径和持续学习激励

---

## 3. 核心业务闭环

平台核心学习闭环为：

**水平测试 → 分级判断 → 单词/句型学习 → 场景对话训练 → 模拟考试 → 学习反馈 → 下一轮学习**

要求：
- 每个模块都要有明确输入与输出
- 测试结果必须承接后续学习
- 模考结果必须能够回流补强学习内容

---

## 4. 功能需求

### 4.1 用户注册 / 登录

| 优先级 | 功能 | 描述 |
|--------|------|------|
| P0 | 用户注册 | 创建账号并进入学习系统 |
| P0 | 用户登录 | 登录后获取个人学习数据 |
| P0 | 登录态管理 | 维持会话与鉴权 |
| P1 | 用户画像 | 记录学习目标、水平、行业背景 |

### 4.2 英语水平测试

| 优先级 | 功能 | 描述 |
|--------|------|------|
| P0 | 测试引导 | 首次进入产品时引导用户完成测试 |
| P0 | 分级测试 | 支持基础能力和商务语境能力评估 |
| P0 | 自动评分 | 自动产出测试分数 |
| P0 | 等级判定 | 输出初级/中级/高级结果 |
| P0 | 测试报告 | 输出优势、薄弱项和学习建议 |
| P1 | 重新测试 | 支持后续重新评估水平 |

### 4.3 商务英语单词学习

| 优先级 | 功能 | 描述 |
|--------|------|------|
| P0 | 分级词汇 | 按初级/中级/高级提供词汇 |
| P0 | 商务主题词汇 | 按会议、邮件、谈判等分类 |
| P0 | 单词详情 | 提供释义、音标、商务例句、搭配 |
| P0 | 学习状态 | 标记已学/待复习/不熟悉 |
| P1 | 复习机制 | 支持错词回顾和重复记忆 |

### 4.4 商务英语句型学习

| 优先级 | 功能 | 描述 |
|--------|------|------|
| P0 | 句型分类 | 按会议、汇报、谈判、邮件等分类 |
| P0 | 场景说明 | 每个句型附带使用场景和功能类型 |
| P0 | 示例表达 | 给出句型示例和替换位 |
| P0 | 学习状态 | 支持记录学习进度 |
| P1 | 收藏/复习 | 支持收藏和复习高频句型 |

### 4.5 分层场景对话训练

| 优先级 | 功能 | 描述 |
|--------|------|------|
| P0 | 场景列表 | 展示可训练场景 |
| P0 | 场景背景与目标 | 明确本轮训练任务 |
| P0 | 多轮对话 | 支持用户与系统连续交互 |
| P0 | AI反馈 | 给出纠错、优化建议和推荐表达 |
| P0 | 训练记录 | 保存本次对话过程 |
| P1 | 对话复盘 | 输出本轮表现总结 |
| P1 | 难度分层 | 不同等级用户匹配不同场景 |

### 4.6 模拟考试

| 优先级 | 功能 | 描述 |
|--------|------|------|
| P0 | 模考入口 | 支持进入模拟考试 |
| P0 | 试卷选择 | 支持按等级选择题目 |
| P0 | 作答过程 | 支持完整答题流程 |
| P0 | 自动评分 | 客观题自动评分 |
| P0 | 成绩结果 | 输出总分、弱项和建议 |
| P1 | 错题沉淀 | 将错误题目纳入后续补强 |

### 4.7 学习记录与反馈

| 优先级 | 功能 | 描述 |
|--------|------|------|
| P0 | 学习记录 | 记录单词、句型、场景、模考结果 |
| P0 | 学习进度汇总 | 展示阶段性学习进度 |
| P1 | 学习看板 | 汇总用户当前状态和下一步建议 |
| P1 | 弱项识别 | 根据学习和模考识别短板 |

### 4.8 商务英语数据采集与内容生产

| 优先级 | 功能 | 描述 |
|--------|------|------|
| P0 | 内容来源管理 | 管理公开资料、自建内容、AI生成内容 |

| P1 | 数据质量监控 | 监控重复率、通过率、质量分布 |

### 4.10 积分系统（后续）

| P2 | 激励玩法 | 连续学习奖励、成就体系 |

## 5. 非功能需求

### 5.1 性能
- 核心页面响应时间应控制在可接受范围内
- 水平测试、对话训练、模考流程需稳定流畅
- 关键数据保存不得明显卡顿

### 5.2 可靠性
- 用户学习记录、测试结果、模考成绩需可靠落库
- 支持异常场景提示和恢复
- 关键流程支持重试或补偿机制

### 5.3 安全性
- 用户账号与学习数据安全存储
- 敏感数据不明文存储
- 内容审核与发布需保留操作记录

### 5.4 可扩展性
- 支持后续扩展更多 BEC 等级内容
- 支持推荐、积分、错题本等能力扩展
- 支持 AI 反馈增强与学习报告增强

## 6. 技术约束

- 技术栈：Python 3.10+
- 后端：FastAPI
- 前端：React + TypeScript
- 数据库：SQLite（开发）/ PostgreSQL（生产）
- 缓存：内存缓存 / Redis（扩展）
- 向量检索：ChromaDB / Qdrant（后续）
- 部署：Docker Compose / 云部署

## 8. 验收标准

### 8.1 第一阶段（MVP）
- [ ] 用户可以成功注册/登录
- [ ] 用户可以完成一次水平测试并获得等级结果
- [ ] 用户可以进入单词和句型学习内容
- [ ] 用户可以完成至少一个场景训练
- [ ] 用户可以完成至少一次模拟考试
- [ ] 学习记录可保存并查看
- [ ] 内容可完成采集、结构化、审核和入库

### 8.2 第二阶段
- [ ] 测试结果能驱动推荐内容
- [ ] 模考结果能回流补强学习内容
- [ ] 对话复盘与学习反馈增强
- [ ] 数据质量监控上线

### 8.3 第三阶段
- [ ] 推荐系统完善
- [ ] 积分和激励体系上线
- [ ] 学习报告增强
- [ ] 内容生产与运营机制成熟
